The web is evolving

Today we're passing from the traditional server app with generated with which a user dialogs through forms to full front-end applications, in which the server becomes a data persistence structure.

Websites used to be sequences of webpages loading in the browser.

Today we're starting to have full featured single-page web applications.

This evolution brings many advantages

Single-page applications offer an experience that is very similar to desktop applications.

Screen transitions can be very fast and smooth. The application is way more reactive, and everything can be connected.
Data persistence is well separated from the interface, and if well made can be reused in other types of interfaces, like mobile applications.

But it can also bring some disadvantages

As the app gains features, the front-end code can quickly become unstructured and difficult to maintain.
This lack of structure can also bring problems when the developer wants to reuse features. We find ourselves making copy/paste of small code portions, which is kind of messy.
